The Steam Content Limiter

This Google App Engine application provides a few small services for the benefit of the Steam Content Server Limiter application, a small helper for customers of ISPs which have strict bandwidth caps. The helper will force Steam to only download from ISP-provided servers to prevent Steam content being charged against the bandwidth cap.

This kind of arrangement is common in Australia and New Zealand where the operating costs of the trans-oceanic fiber cables are borne by ISPs who then use data caps to manage the load their customers offer. Since Steam itself has no support for limiting which servers it downloads from, this third-party helper does the job.

The services here for the Steam applet to use are:

  • /latest to show the most recent app version
  • /download to access the download page for the latest version
  • /ispname for a best guess at which supported ISP you are hosted on
  • /filterrule which returns a filter rule for your ISP as reported in /ispname
  • /feedback which provides a simple, anonymous feedback mechanism to the project author.
  • /uploadrule which provides a simple, anonymous method of contributing custom rule suggestions for the filter tools

These services are primarily intended to allow the installer for the Steam Limiter to automatically determine which of the supported ISPs you are a customer of so that the correct filter is used by default, and to help the Limiter to automatically check for new versions and assist in downloading the installer for the latest version

No personally identifiable information is retained by this service, although for the purposes of debugging the /ispname and /filterip services the external IP address of the visiting host is written to the App Engine administrative log (just as any webserver does).

This service uses a subset of the GeoLite data created by MaxMind, available from http://www.maxmind.com/ to determine which supported ISP you are visiting from.
